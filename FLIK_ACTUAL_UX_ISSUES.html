🔴 CRITICAL ISSUES (Weeks 1-3, 100-120 hours)

CRITICAL #1: Button Label Inconsistency

Impact: HIGH Effort: 2-3 days Fixability: EASY

Problem:

Same action has different labels across products, confusing users every time they switch.

Action Dashboard Checkout App O2O
Continue/Next "Lanjutkan" "Lanjut" "Selanjutnya" "?"
Save "Simpan" "Bayar" "Konfirmasi" "?"
Cancel "Tidak" "Batal" "Kembali" "?"
Real User Scenario:
User switches between Dashboard and Checkout. Same action ("continue") has different button text. User wastes cognitive effort relearning button labels for each product. Power users (who use all products) are especially frustrated.
Impact:
- Reduced cognitive efficiency
- Perceived lack of polish
- Increased support burden (FAQ: "where do I click to continue?")
- Brand consistency loss

Why It's a UX Issue (Not Documentation):

This isn't about documenting what buttons exist - it's about users seeing inconsistent interaction patterns. Documentation won't solve this; standardizing the actual buttons will.

Solution:

  • ✓ Choose standard labels: "Lanjut" (Continue), "Simpan" (Save), "Batal" (Cancel), "Tambah" (Add)
  • ✓ Update all 50+ buttons across 4 products
  • ✓ Enforce in code (component library)
  • ✓ Add to design guidelines

CRITICAL #2: Authentication Method Inconsistency

Impact: CRITICAL Effort: 3-4 weeks Fixability: MEDIUM

Problem:

Different products use different authentication methods, creating confusion and security friction.

Product Authentication Method Requires Account
Merchant Dashboard Email + Password Yes (separate)
Checkout WhatsApp OTP No (guest option)
Shopper App WhatsApp OTP Yes (required)
O2O Credentials (unclear) Yes (POS staff)
Real User Scenario:
User is a merchant using Dashboard. Password gets forgotten. User tries to use Checkout with same credentials → Doesn't work (because Checkout uses WhatsApp OTP). User is confused: "Do I have one account or multiple?" Creates support burden and lost sales during account recovery.
Impact:
- Credential management burden
- Forgotten password = lost transactions
- Security anxiety (different methods = unclear trust)
- Higher support load
- Potential account access issues

Why It's a UX Issue (Not Documentation):

Users experience different auth flows. Documentation won't make them consistent; unifying the actual authentication system will.

Solution:

  • ✓ Standardize to WhatsApp OTP across consumer products (Checkout, App)
  • ✓ Keep email/password for Dashboard (admin use case)
  • ✓ Create unified account system
  • ✓ Implement SSO (Single Sign-On) where applicable

CRITICAL #3: Navigation Pattern Inconsistency

Impact: MEDIUM-HIGH Effort: 2-3 weeks Fixability: MEDIUM

Problem:

Each product uses completely different navigation approach. Power users (who use multiple products) are disoriented.

Product Navigation Type User Experience
Dashboard Left sidebar Persistent, clear hierarchy
Checkout Modal-based steps Linear, no lateral navigation
App Bottom tabs Mobile-optimized, limited options
O2O Dashboard buttons Flat structure, limited navigation
Real User Scenario:
Merchant uses Dashboard (sidebar nav), then opens Checkout (modal nav), then uses Shopper App (tabs). Back button behavior is inconsistent. User wastes cognitive energy relearning navigation for each product. Multi-platform workflow is painful.
Impact:
- Reduced efficiency for power users
- Higher cognitive load
- Back button behavior unpredictable
- Momentum lost switching between products

Solution:

  • ✓ Use consistent navigation pattern across products (recommend: sidebar for desktop, bottom tabs for mobile)
  • ✓ Standardize back button behavior
  • ✓ Add breadcrumbs for clarity
  • ✓ Maintain consistent state when switching products

CRITICAL #4: Payment Success Confirmation Unclear

Impact: CRITICAL Effort: 1-2 weeks Fixability: EASY

Problem:

After user clicks "pay", it's unclear if payment succeeded or is still processing. This creates anxiety and abandoned transactions.

Real User Scenario:
User clicks "Bayar Sekarang" (Pay Now) on Checkout. Page shows... nothing? Or loading spinner? Or redirects? User doesn't know if payment went through. User clicks "Pay Now" again (double payment). Or leaves page thinking transaction failed.
Impact:
- High checkout abandonment
- Duplicate payments
- Customer support load
- Lost revenue
- Negative user sentiment

Solution:

  • ✓ Show clear loading state with message: "Processing payment..."
  • ✓ Disable button during processing
  • ✓ Show success screen clearly: "Payment successful!"
  • ✓ Show clear error if payment fails
  • ✓ Add email confirmation immediately

🟠 HIGH PRIORITY ISSUES (Weeks 3-4, 80-100 hours)

HIGH #1: Form Validation Error Messages Inconsistent

Impact: MEDIUM Effort: 1-2 weeks Fixability: EASY

Problem:

Different error messages for same validation failures. Users confused about what's wrong.

Example:
Email field in Dashboard says: "Invalid email format"
Email field in Checkout says: "Email address not valid"
Email field in App says: "Please enter valid email"
User doesn't understand the error varies by product.

Solution:

  • ✓ Create error message templates
  • ✓ Standardize across all products
  • ✓ Make messages helpful: show what's wrong + how to fix

HIGH #2: Mobile Responsiveness Issues

Impact: MEDIUM Effort: 2-3 weeks Fixability: MEDIUM

Problem:

Some products have poor mobile experience. Touch targets too small. Layouts break on tablets.

Impact:

  • Users abandon on mobile (bounce rate up)
  • Difficult to use on tablets
  • Touch targets hard to hit (48px minimum)

HIGH #3: Session/Logout Inconsistency

Impact: HIGH (Security) Effort: 1-2 weeks Fixability: MEDIUM

Problem:

Different products handle session timeouts differently. User logs out in one product, still logged in elsewhere. Security concern.

HIGH #4: Shopper App Deep Linking Issues

Impact: MEDIUM Effort: 2-3 weeks Fixability: MEDIUM

Problem:

Can't deep link to specific screens in app. Modals don't integrate well with browser back button.

HIGH #5: O2O Payment Feedback Missing

Impact: MEDIUM Effort: 1-2 weeks Fixability: EASY

Problem:

POS system doesn't give clear feedback when QRIS/EDC payment is processing. Cashier doesn't know if customer should wait or try again.

🟡 MEDIUM PRIORITY ISSUES (Weeks 4-5, 40-50 hours)

MEDIUM #1: Terminology Inconsistency

Same concept called different names: "Pesanan" vs "Order", "Pengiriman" vs "Shipping", etc.

MEDIUM #2: Notification Handling

Different products notify users differently. No consistency in where/how notifications appear.

MEDIUM #3: Empty State Messaging

When lists are empty, some products show helpful message + action, others show nothing.
